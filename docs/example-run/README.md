# Example Pipeline Run — Wide World Importers

> **These are sample outputs only.**
>
> Everything in this directory was generated by running the full accelerator
> pipeline against the
> [Microsoft Wide World Importers sample corpus](https://github.com/microsoft/sql-server-samples/tree/master/samples/databases/wide-world-importers)
> as part of initial testing and validation. They are committed here so
> engineers and evaluators can see what the pipeline produces **without
> having to run it themselves first**.
>
> When you run the pipeline against your own source repo, your outputs will
> differ — object names, counts, complexity scores, converted SQL, and
> Workflow tasks will all reflect your actual codebase, not WWI.

---

## What's in here

### `pipeline-outputs/` — Analysis outputs

Generated by `run_analysis.py` → `run_impact_analysis.py` → `run_target_state_design.py`.

| File / folder | What it contains |
|---|---|
| `inventory.json` | Every SQL/SSIS object found: type, schema, name, dependencies, complexity |
| `unsupported_objects.json` | Objects the parser couldn't classify |
| `dependencies.json` | Directed dependency graph with topological order |
| `current_state_documentation.md` | Human-readable inventory summary |
| `impact_analysis.md` | 12-dimension risk assessment per object |
| `object_complexity_scores.json` | Per-object complexity score and classification |
| `medallion_mapping.csv` | Source object → target Unity Catalog FQN mapping |
| `target_state_architecture.md` | Full medallion architecture design with rationale |
| `manual_intervention_list.md` | Objects requiring manual work before deployment |
| `test_matrix.csv` | Scenario-driven test matrix |
| `migration_risk_register.csv` | Risk register sorted by severity |

### `pipeline-converted/` — Conversion outputs

Generated by `run_conversion.py` → `run_ssis_conversion.py`.

| File / folder | What it contains |
|---|---|
| `databricks_sql/` | Databricks SQL DDL — one `.sql` file per converted object, organised by layer (bronze/silver/gold) |
| `pyspark/` | PySpark modules for procedures and orchestration logic |
| `review_required/` | One `.md` per object flagged for manual review — includes original DDL and suggested approach |
| `conversion_manifest.json` | Machine-readable record of every object's conversion outcome |
| `conversion_decisions.md` | Rationale for type mappings and conversion choices |
| `workflow_spec.json` | Databricks Workflow job spec generated from the SSIS package |
| `ssis_conversion_report.md` | Summary of the SSIS → Workflow conversion |

### Other files (validation and review)

| File | What it contains |
|---|---|
| `validation_summary.md` | Pass/partial/fail verdict for each of the 12 validation steps |
| `validation_results.json` | Machine-readable validation results |
| `failed_cases.json` | Detailed evidence for every non-passing step |
| `adversarial_review.md` | Self-critical review: silent failures, naive translations, edge cases |
| `recommended_backlog.md` | Ranked list of issues found during validation |
| `remediation_backlog.csv` | Issues found during the adversarial review and their status |

## How to regenerate these outputs

To run the pipeline against the WWI corpus yourself and get fresh outputs:

```bash
# Clone the WWI sample (sparse checkout)
git clone --no-checkout https://github.com/microsoft/sql-server-samples.git ../sql-server-samples
cd ../sql-server-samples
git sparse-checkout init --cone
git sparse-checkout set samples/databases/wide-world-importers
git checkout main
cd ../sql-ssis-to-databricks-accelerator

# Run the full pipeline
python accelerator/cli.py \
    --oltp-dir ../sql-server-samples/samples/databases/wide-world-importers/wwi-ssdt/wwi-ssdt \
    --dw-dir   ../sql-server-samples/samples/databases/wide-world-importers/wwi-dw-ssdt/wwi-dw-ssdt \
    --ssis-dir ../sql-server-samples/samples/databases/wide-world-importers/wwi-ssis/wwi-ssis
```

Outputs land in `outputs/` and `output/` at the repo root (gitignored, so
they won't overwrite what's committed here).
