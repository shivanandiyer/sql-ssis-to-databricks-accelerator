## 10. Observed Naming Conventions

### OLTP (WideWorldImporters)
- Schema prefix matches business domain: `Sales`, `Purchasing`, `Warehouse`, `Application`
- Table names are PascalCase nouns (`Orders`, `Customers`, `StockItems`)
- Primary keys follow pattern `<EntityName>ID` (e.g. `OrderID`, `CustomerID`)
- Temporal tables have paired `<Table>_Archive` history tables
- Integration schema holds source-extraction stored procs (`Get<Entity>Updates`)
- Views are in `Website.*` and `WebApi.*` schemas

### DW (WideWorldImportersDW)
- Schemas map directly to medallion layers: `Integration` → Bronze staging,
  `Dimension` → Silver, `Fact` → Gold
- Surrogate keys: `<Entity> Key` (space-separated, e.g. `City Key`)
- Source system natural keys: `WWI <Entity> ID`
- SCD Type 2 markers: `Valid From`, `Valid To` (datetime2), `Lineage Key`
- Fact tables partitioned on date key (`PS_Date` partition scheme)
- Staging tables: `Integration.<Entity>_Staging` — truncate-and-load pattern

### SSIS
- One package: `DailyETLMain.dtsx`
- One Sequence Container per entity: `Load <Entity> Dimension` / `Load <Entity> Fact`
- Consistent 5-step pattern per entity:
  1. `Get Last <Entity> ETL Cutoff Time`
  2. `Truncate <Entity>_Staging`
  3. `Extract Updated <Entity> Data to Staging` (Data Flow)
  4. `Get Lineage Key`
  5. `Migrate Staged <Entity> Data`
- Variables: `LastETLCutoffTime`, `TargetETLCutoffTime`, `LineageKey`, `TableName`
